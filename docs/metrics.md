# 指标解释

更新时间：2026-05-11

## 口径说明

系统现在同时服务两类场景：

- 通用信息茧房推演：观察观点如何传播、极化和纠偏；
- 新闻事件推演：结合新闻可信度，判断“相信/拒绝”是否构成“误信/正确认知”。

因此不要只看一个字段。演示时建议按以下顺序解释：

1. 先解释 `opinion` 表示 Agent 对当前新闻的态度。
2. 再解释新闻可信度如何决定“相信新闻”是否等于“正确认知”。
3. 最后解释误信率、正确认知率、极化和沉默等群体结果。

## 观点与可信度

### `opinion`：态度值

范围：`-1` 到 `+1`。

| 区间 | 含义 |
| --- | --- |
| `< -0.2` | 倾向拒绝当前新闻或倾向负向判断 |
| `-0.2 ~ 0.2` | 中立、观望、摇摆 |
| `> 0.2` | 倾向相信当前新闻或倾向正向判断 |

### `news_credibility`：新闻可信度

取值：

- `高可信`
- `低可信`
- `不确定`

可信度影响后验指标含义：

| 新闻可信度 | 相信新闻 | 拒绝新闻 |
| --- | --- | --- |
| 高可信 | 正确认知 | 误信 |
| 低可信 | 误信 | 正确认知 |
| 不确定 | 相信 | 拒绝 |

## 群体核心指标

| 字段 | 前端名称 | 解释 |
| --- | --- | --- |
| `believe_rate` | 相信率 | `opinion > 0` 的 Agent 比例 |
| `reject_rate` | 拒绝率 | `opinion < 0` 的 Agent 比例 |
| `uncertain_rate` | 不确定率 | 态度接近中立的比例 |
| `mislead_rate` | 误信率 | 按新闻可信度后验判定后，处在错误认知一侧的比例 |
| `correct_rate` | 正确认知率 | 按新闻可信度后验判定后，处在正确认知一侧的比例 |
| `negative_belief_rate` | 误信率（兼容） | 兼容旧字段，当前与误信口径对齐 |
| `positive_belief_rate` | 正确认知率（兼容） | 兼容旧字段，当前与正确认知口径对齐 |
| `avg_opinion` | 平均态度 | 群体态度平均值 |
| `polarization_index` | 极化指数 | 群体观点分歧程度 |
| `silence_rate` | 沉默率 | 选择不表达观点的 Agent 比例 |

## 深度指标

| 字段 | 前端名称 | 解释 |
| --- | --- | --- |
| `deep_believe_rate` | 深度相信率 | 相信且信念强度较高的比例 |
| `deep_reject_rate` | 深度拒绝率 | 拒绝且信念强度较高的比例 |
| `deep_negative_rate` | 深度误信率（兼容） | 兼容口径 |
| `deep_positive_rate` | 深度正确认知率（兼容） | 兼容口径 |
| `weighted_believe_index` | 加权相信指数 | 将相信程度和信念强度合并后的指数 |
| `weighted_negative_index` | 加权误信指数（兼容） | 兼容口径 |

## 双层网络指标

| 字段 | 前端名称 | 解释 |
| --- | --- | --- |
| `public_negative_rate` | 公域误信率 | 公域网络中的误信口径比例 |
| `public_positive_rate` | 公域正确认知率 | 公域网络中的正确认知口径比例 |
| `private_negative_rate` | 私域误信率 | 私域社群中的误信口径比例 |
| `private_positive_rate` | 私域正确认知率 | 私域社群中的正确认知口径比例 |
| `num_communities` | 社群数 | 私域网络中的社群数量 |
| `num_influencers` | 影响者数 | 公域高影响力节点数量 |

旧字段仍会返回：

- `rumor_spread_rate` = `negative_belief_rate`
- `truth_acceptance_rate` = `positive_belief_rate`
- `public_rumor_rate` = `public_negative_rate`
- `public_truth_rate` = `public_positive_rate`
- `private_rumor_rate` = `private_negative_rate`
- `private_truth_rate` = `private_positive_rate`

## Agent 个体指标

点击信息传播网络节点后，前端通过 `/api/agent/{id}/inspect` 展示个体指标。

| 字段 | 前端名称 | 解释 |
| --- | --- | --- |
| `belief_strength` | 信念强度 | 数值越高，观点越不容易改变 |
| `susceptibility` | 易感性 | 数值越高，越容易受外部信息和邻居观点影响 |
| `influence` | 影响力 | 数值越高，越容易影响其他 Agent |
| `fear_of_isolation` | 孤立担忧 | 数值越高，越担心表达少数观点后被孤立 |
| `conviction` | 立场坚定度 | 数值越高，越坚持当前判断 |
| `is_silent` | 是否沉默 | 当前是否选择不表达观点 |
| `perceived_climate` | 邻居舆论气候 | 周边邻居的态度分布和沉默情况 |

### 邻居舆论气候

`perceived_climate` 的扁平结构：

| 字段 | 解释 |
| --- | --- |
| `total` | 邻居数量 |
| `pro_rumor_ratio` | 邻居中倾向误信/负向的比例 |
| `pro_truth_ratio` | 邻居中倾向正确认知/正向的比例 |
| `neutral_ratio` | 邻居中中立比例 |
| `silent_ratio` | 邻居中沉默比例 |
| `avg_opinion` | 邻居平均态度 |

## 现实画像字段

现实画像和用户资料画像都会附加在 Agent 的 `realistic_profile` 中。

| 字段 | 前端名称 | 解释 |
| --- | --- | --- |
| `name` | 姓名 | 演示验证用，当前不脱敏显示 |
| `role_label` | 角色标签 | 资历、职称、管理职责、研究方向的组合描述 |
| `department` | 部门 | 所属部门 |
| `specialty` | 研究方向 | 专业背景 |
| `title` | 职称/岗位 | 工作簿中的职称或岗位信息 |
| `seniority_label` | 资历层级 | 青年、中坚、资深等 |
| `community_id` | 社群编号 | 按研究方向和部门映射的私域社群 |
| `is_influencer` | 是否影响者 | 影响力高或有管理职责时为 true |
| `generation_trace` | 数值来源 | 初始参数如何生成 |

## 数值来源

`generation_trace` 用于解释现实画像的初始数值，不应该直接把英文变量暴露给客户。建议前端折叠展示。

### 来源字段

| 字段 | 中文解释 |
| --- | --- |
| `source` | 来源：`workbook` 表示来自工作簿；`user_profile_library` 表示来自用户资料库；`synthetic_roster` 表示合成兜底画像；`cached_profile` 表示缓存 |
| `inputs` | 生成数值时使用的输入，例如年龄段、工龄段、职称、职务、学历、研究方向 |
| `derived` | 派生变量，例如资历分、社群编号、是否影响者 |
| `metrics` | 最终生成的初始指标 |
| `formulas` | 中文规则解释 |

### 派生变量

| 内部字段 | 前端名称 | 客户解释 |
| --- | --- | --- |
| `seniority_score` | 资历分 | 根据年龄段、工龄、本单位年限、职称和管理职责估算的资历强度，范围 0 到 1 |
| `community_id` | 社群编号 | 根据研究方向或部门归入的私域社群 |
| `is_influencer` | 是否影响者 | 影响力较高或具有管理职责的节点 |

### 指标生成规则

| 指标 | 中文解释 |
| --- | --- |
| 初始态度 | 以中性为中心叠加小幅随机扰动，避免预设真实人员立场 |
| 信念强度 | 资历分越高通常越稳定，再叠加少量扰动 |
| 影响力 | 资历分和行政职务共同影响，最后限制在 0.1 到 1.0 |
| 易感性 | 资历分越高通常越不容易受影响；博士学历会略微降低易感性 |
| 孤立担忧 | 行政或党内职务会略微提高对被孤立的敏感度 |
| 立场坚定度 | 资历分越高通常越坚定，再叠加少量扰动 |

术语替换：

| 英文/代码词 | 应展示为 |
| --- | --- |
| `clip` | 限制在合理范围内 |
| `seniority_score` | 资历分 |
| `N(...)` | 小幅随机扰动 |
| `workbook` | 人员工作簿 |
| `user_profile_library` | 用户资料库 |
| `synthetic_roster` | 合成兜底画像 |

用户资料画像的额外输入：

| 字段 | 解释 |
| --- | --- |
| `source_file` | 该成员画像主要来自哪个本地资料文件 |
| `notes_excerpt` | 资料简介或正文摘录，最多用于解释画像生成依据 |

资料丰富度会轻微影响信念强度、影响力、易感性和立场坚定度：资料越完整，系统越少依赖默认值；但初始态度仍围绕中性生成，不把文本内容直接判定为真实立场。

## 预测指标

`GET /api/prediction` 和 `/api/prediction/trajectory` 返回预测相关指标。

| 字段 | 解释 |
| --- | --- |
| `expected` | 期望情景 |
| `optimistic` | 乐观情景 |
| `pessimistic` | 悲观情景 |
| `confidence` | 置信度 |
| `recommendation` | 干预建议 |
| `trajectory` | 未来多步轨迹 |

历史状态少于 3 条时，预测接口返回 `available=false`。

## 风险指标

风险预警综合以下信号：

- 误信率是否超过阈值；
- 极化指数是否过高；
- 沉默率是否过高；
- 误信率是否持续上升；
- 权威回应后是否未产生纠偏效果；
- 公域和私域是否出现明显分裂。

常用解释：

| 风险 | 含义 |
| --- | --- |
| 低风险 | 主要指标处于可控区间 |
| 中风险 | 误信、极化或沉默压力已显现 |
| 高风险 | 误信或极化形成主导，需要立即干预 |

## 报告指标

基础报告和智库专报会使用统一报告口径：

- 高可信新闻：相信计为正确认知，拒绝计为误信；
- 低可信新闻：相信计为误信，拒绝计为正确认知；
- 不确定新闻：同时保留相信/拒绝和误信/正确认知解释；
- 现实画像和用户资料画像样本在报告中优先显示姓名、角色、部门和研究方向；
- 典型 Agent 样本会展示观点变化、信念强度、易感性、推理和评论。

## 演示建议

向客户解释时建议使用中文指标名，不直接展示内部字段名：

| 客户问题 | 推荐解释 |
| --- | --- |
| “这个人的影响力怎么来的？” | 由资历分、职务和角色共同估算，并限制在合理范围 |
| “为什么这个人初始态度不是 0？” | 为避免所有人完全相同，系统围绕中性加入很小扰动，不代表真实立场 |
| “资历分是什么？” | 综合年龄段、工龄、本单位年限、职称和管理职责得到的归一化分数 |
| “为什么姓名要显示？” | 用于演示时验证具体节点与微观行为是否对应；敏感字段仍被排除 |
| “用户资料画像是不是每次重新生成？” | 不是。离线构建后会读取本地缓存，只有重新点击构建或刷新缓存才重新解析资料 |
