# 推演机制

更新时间：2026-05-24

## 三个维度

系统的推演配置可以理解为三个独立维度：

| 维度 | 选项 | 说明 |
| --- | --- | --- |
| 推演任务 | `sandbox` / `news` | 沙盘模式偏机制实验；新闻模式偏事件与风险评估 |
| 决策引擎 | 数学模型 / LLM | 数学模型快速、可复现；LLM 提供更丰富的微观决策链 |
| 人设来源 | 理论人设 / 内置现实画像 / 用户资料画像 | 理论人设适合通用机制；现实画像适合近身组织样本演示；用户资料画像适合按客户资料离线构建样本 |

默认网络为双层网络：公域无标度网络 + 私域社群网络。

## 观点表示

每个 Agent 有一个 `opinion`，范围为 `[-1, 1]`：

```text
-1                         0                         +1
完全拒绝 / 负向           中立 / 摇摆                完全相信 / 正向
```

新闻推演中，系统会根据新闻可信度把“相信/拒绝”后验解释为“误信/正确认知”：

| 新闻可信度 | 相信新闻 | 拒绝新闻 |
| --- | --- | --- |
| 高可信 | 正确认知 | 误信 |
| 低可信 | 误信 | 正确认知 |
| 不确定 | 保留相信/拒绝口径，误信结论仅作模拟参考 |

因此前端和报告中应同时注意：

- `believe_rate` / `reject_rate`：直接表示对当前新闻的态度；
- `mislead_rate` / `correct_rate`：结合可信度后的研判口径；
- `negative_belief_rate` / `positive_belief_rate`：兼容字段。

## 理论人设

理论人设由系统根据内置类型和随机扰动生成。它保留了可解释的社会心理变量：

- 信念强度；
- 易感性；
- 影响力；
- 孤立担忧；
- 立场坚定度；
- 沉默状态；
- 邻居舆论气候。

理论人设适合回答“如果茧房更强、权威回应更晚、逆火更大，会发生什么”这类机制问题。

## 现实组织画像

现实画像模式用于回答“我附近的真实组织成员可能如何看待某个新闻”这类问题。

当前支持：

```text
shass_news_institute
```

加载后，系统用工作簿或缓存中的组织信息构造 27 个 Agent。使用的信息包括：

- 姓名；
- 部门；
- 职称/岗位；
- 行政或党内职务；
- 学历/学位；
- 年龄段、工龄段、本单位年限段；
- 研究方向。

不使用或不输出：

- 身份证号；
- 手机、邮箱、联系地址；
- 健康、婚姻、血型、户口、出生日期等敏感字段。

### 初始数值如何产生

现实画像的初始数值来自 `generation_trace`：

1. 从工作簿读取非敏感输入。
2. 根据年龄段、工龄、职称、职务等计算“资历分”。
3. 根据资历分、行政职务、学历学位和小幅随机扰动生成各项指标。
4. 把每个指标限制在合理范围内，避免极端初始值。

主要规则：

| 指标 | 生成逻辑 |
| --- | --- |
| 初始态度 | 以中性为中心叠加小幅扰动，避免预设真实人员立场 |
| 信念强度 | 资历分越高通常越稳定，再叠加小幅扰动 |
| 影响力 | 资历分和行政职务共同提高影响力 |
| 易感性 | 资历分越高通常越不易受影响；博士学历会略微下调易感性 |
| 孤立担忧 | 行政或党内职务会略微提高对被孤立的敏感度 |
| 立场坚定度 | 资历分越高通常越坚定 |
| 社群编号 | 根据研究方向和部门归入不同社群 |
| 是否影响者 | 影响力较高或有管理职责时标记为影响者 |

前端“数值来源”应折叠展示，避免微观行为透视过载。鼠标悬停在指标名上时，可以展示对应计算规则。

## 用户资料画像

用户资料画像用于回答“我能否用客户提供的表格、文章、报道和内部材料构建一个可复用群体”这类问题。它与内置现实画像共享推演引擎和微观透视结构，但来源由固定工作簿改为本地资料库。

构建流程：

1. 在前端“样本画像 -> 自定义资料画像 -> 资料库构建”填写画像 ID 和显示名称。
2. 上传 `CSV/TSV/JSON/JSONL/TXT/MD/Markdown` 资料；安装 `pandas/openpyxl` 后也支持 Excel。
3. 点击“离线构建画像”，后端抽取姓名、部门、职称、研究方向、年龄、工龄和简介。
4. 生成 `data/realistic_profiles/<profile_id>.sanitized.json` 缓存。
5. 后续推演选择该画像即可复用缓存，不需要重新解析原始资料。

默认资料目录：

```text
data/user_profiles/<profile_id>/sources/
```

用户资料画像的数值来源为 `generation_trace.source = user_profile_library`。初始指标仍遵循“中性态度 + 资历/角色/资料丰富度估算 + 小幅稳定扰动”的规则，不把资料直接等同于真实态度。

## 数学模型

数学模型用公式更新观点，适合快速推演和可复现实验。

核心影响项：

```text
new_opinion = f(
  社交影响,
  算法茧房,
  权威回应,
  逆火效应,
  群体极化,
  沉默螺旋,
  知识图谱影响
)
```

### 社交影响

Agent 会受到邻居观点影响。邻居影响力越高，对目标 Agent 的权重越大。

```text
social_influence = 加权邻居观点平均值
```

### 算法茧房

`cocoon_strength` 越高，推荐内容越强化既有观点，群体更容易走向同温层。

```text
cocoon_strength = 0：无茧房强化
cocoon_strength = 1：高度同质化推荐
```

### 权威回应

`response_delay` 控制权威回应何时介入，`response_credibility` 和 `authority_factor` 决定回应强度，`backfire_strength` 表示部分 Agent 因抵触而更坚定原观点。

### 沉默螺旋

当 Agent 感知到邻居观点与自己明显相反，且自身孤立担忧较高，就可能选择沉默。沉默影响表达和传播，而不是直接删除观点。

### 群体极化

同质社群内部讨论会强化群体主流观点，使两端距离扩大。

### 知识图谱影响

新闻事件被解析为实体、关系、摘要、情感和可信度后，会影响 Agent 对事件的理解。实体重要度、关系方向和新闻可信度会进入演化和报告逻辑。

## LLM 驱动

LLM 模式中，每个 Agent 在每一步收到结构化上下文：

- 自己的人设和现实画像；
- 当前观点、信念强度、易感性、影响力；
- 邻居观点和舆论气候；
- 收到的新闻、权威回应和事件图谱；
- 双层网络中的公域/私域环境。

LLM 返回：

- 新观点；
- 情绪；
- 行动；
- 可能评论；
- 推理过程；
- 原始响应。

这些内容会进入 Agent 快照，供 `/api/agent/{id}/inspect` 和前端微观行为透视展示。

## 双层网络

### 公域网络

公域网络使用无标度结构，模拟公开平台传播：

- 少数高影响力节点拥有更多连接；
- 传播速度更快；
- 更适合观察议题扩散和影响者效应。

### 私域网络

私域网络使用社群结构，模拟熟人群组：

- 社群内部连接密集；
- 跨社群传播相对更难；
- 更适合观察沉默、同温层和局部极化。

## 事件注入

事件注入有两种方式：

| 模式 | `skip_parse` | 说明 |
| --- | --- | --- |
| 完整解析 | `false` | 调用 LLM 提取知识图谱，适合首次注入和深度分析 |
| 快速注入 | `true` | 跳过解析，构造简化图谱，适合推演中快速施加冲击 |

事件来源：

- `public`：注入公域；
- `private`：注入私域；
- 其他情况可视为全域。

如果推演尚未开始，事件会暂存并在下一次启动时自动注入。

## 预测与风险

预测模型需要至少 3 条历史状态。输出包括：

- 期望轨迹；
- 乐观场景；
- 悲观场景；
- 干预建议；
- 风险摘要。

风险预警关注：

- 误信率；
- 极化指数；
- 沉默率；
- 权威回应后是否有效；
- 指标是否持续上升。

## 报告生成

### 基础报告

基础报告由引擎生成，偏推演复盘。它会汇总：

- 参数配置；
- 事件池；
- 新闻可信度口径；
- 知识图谱；
- 趋势变化；
- 权威回应效果；
- 风险等级；
- 典型 Agent 样本；
- 干预建议。

### 智库专报

智库专报由 `AnalystAgent` 通过 LLM 生成，偏面向客户或决策者阅读。当前 prompt 已要求结合：

- 样本画像；
- 新闻内容和可信度；
- 宏观趋势；
- 微观决策切片；
- 风险与建议；
- 现实画像模式下的姓名、角色和专业背景。

## 使用建议

| 目标 | 推荐配置 |
| --- | --- |
| 快速解释模型机制 | 理论人设 + 数学模型 + 单步推演 |
| 演示真实组织近身样本 | `shass_news_institute` + LLM + 双层网络 |
| 按客户资料离线构建群体 | 自定义资料画像 + LLM + 双层网络 |
| 低成本批量实验 | 理论人设 + 数学模型 + 大人口 |
| 深度报告 | LLM + 至少一步推演 + 智库专报 |
| 推演中事件冲击 | WebSocket 自动推演 + `/api/event/airdrop` |
